一句話：記住這位旅客的偏好，再回答行程——先檢索記憶，再呼叫 Qwen，最後寫回。

  • 模式對齊上游 ai_travel_agent_memory，底座改為 Redis + Qwen
  • 記憶按 user_id 隔離，示範帳 demo_traveler
  • 與 CRM／RAG 向量庫分開，不混存

主路徑（文字版）

User 提問
  → search_memories(user_id)
  → Qwen(旅行助理 + 相關記憶)
  → add_memory(user + assistant)
  → 回傳 reply + relevant_memories

演示 60 秒

  1. 上方 #travel 載入示範記憶
  2. 問「京都三天，慢節奏」→ 狀態列顯示用到的記憶條數
  3. 換另一個 user 名 → 記憶箱應為空（隔離）
