① RAG 失敗診斷診所

上游 rag_failure_diagnostics_clinic · 站內演示 #clinic · 源碼 clinic/ + app/clinic.py

一句話： 貼上真實故障描述 → 對照 P01–P12 可複用模式 → 輸出 主因 + 最小結構修復 （不是「換更大模型」或「再塞一點 context」）。

主路徑

使用者貼故障描述
  → 載入 P01–P12 模式庫（system prompt）
  → Qwen 選唯一 primary pattern
  → 可選 secondary + 證據摘錄
  → 最小結構修復建議
  → 可寫 clinic/reports/*.json

本站適配

  • CLI 仍在 clinic/rag_failure_diagnostics_clinic.py
  • Web：POST /api/clinic/diagnose · 繁中輸出
  • LLM：LAN Qwen（OpenAI 相容），非 OpenAI 雲端
  • 內建範例：P01 檢索幻覺、P10 啟動順序、P11 設定漂移、CATCH RAG 案例

P01–P12 速查

ID模式典型症狀
P01檢索幻覺／接地漂移答案與檢索文件矛盾
P02分塊邊界錯誤事實被切碎或截斷
P03嵌入語義錯位向量近、語意遠
P04索引過期／偏斜源已更新索引仍舊
P05查詢改寫／路由錯打到錯誤工具或庫
P06長鏈推理漂移多步後忘約束
P07工具誤用參數錯、無接地
P08會話記憶缺失跨輪丟關鍵事實
P09評測盲區測過、線上翻車
P10啟動依賴未就緒部署初期 5xx
P11設定／密鑰漂移本機可、正式掛
P12多租戶互相踩狀態被覆蓋

演示 60 秒

  1. 打開主站 #clinic
  2. 選「檢索幻覺 (P01)」範例 → 執行診斷
  3. 讀 primary pattern、結構修復建議；可對照本站 RAG 故障再跑 CATCH 範例

② AI 旅遊助理（記憶）

上游 ai_travel_agent_memory · 站內 #travel · 源碼 travel_agent_memory/ + app/travel_memory.py

一句話： 記住這位旅客的偏好，再回答行程 ——先檢索記憶，再呼叫 Qwen，最後把對話寫回（不是裸聊）。

主路徑

User 提問
  → search_memories(user_id, query)
  → Qwen(旅行助理 + 相關記憶)
  → add_memory(user + assistant)
  → 回傳 reply + relevant_memories

本站適配

  • 原版：Streamlit + Mem0 + Qdrant + GPT-4o
  • 本站：Web + Redis list + 關鍵詞檢索 + Qwen
  • 示範帳 demo_traveler：慢旅行、自然／咖啡、火車、堅果過敏
  • 鍵：mkt:travel:mem:{user} · chat log 另存

為什麼改寫棧

原版本站
UIStreamlit#travel 區塊
記憶Mem0Redis list
向量Qdrant演示省略（模式仍「先憶後答」）
LLMGPT-4oQwen :8080
隔離user_id相同

不變的契約：記憶是產品能力，不是模型本身。升格生產可換回向量 Mem0／embedding。

演示 60 秒

  1. 打開 #travel，載入示範記憶
  2. 問「京都三天，慢節奏」→ 狀態列顯示用到的記憶條數
  3. 換另一個 user 名 → 記憶箱應為空（隔離）

③ 對照：為什麼放在同一站

同一套 LAN 基建（Qwen · Redis · FastAPI），演示兩種「可賣的 AI 模式」。

維度 診斷診所 旅遊記憶
產品判斷 模式分診 + 結構修復 長期偏好記憶
輸入 故障敘事（incident） 旅行意圖 + user_id
狀態 可選 JSON 報告 Redis 記憶 list／對話
與站內 RAG 可診斷 RAG 故障 不共用向量 KB
商機鉤子 事故複盤、SRE 工具化 垂直助理（旅遊／保險／客服）
演示錨點 #clinic #travel

系統邊界（文字圖）

瀏覽器
  ├─ #clinic  → POST /api/clinic/*  → Qwen（分診 P01–P12）→ reports/
  ├─ #travel  → POST /api/travel/*  → Redis 記憶 + Qwen 行程
  ├─ #rag     → /api/rag/*          → Ollama embed + KB（獨立）
  └─ CRM 指標 → Postgres catch_crm  （與上述兩 tutorial 無關）

基建：FastAPI :18721 · Redis :6379 · Qwen :8080 · Ollama · Postgres :5432
