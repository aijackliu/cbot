## 一句話

在**個人微信／企業微信**自動**維護客戶訊息** → **識別購買意向** → **轉交銷售**，形成可運轉的成交前閉環。

## 三層架構

| 層 | 作用 | 關鍵產物 |
|----|------|----------|
| 入口層 | 個微 + 企微雙通道 | 統一客戶池 |
| 智能層 | 歸檔、標籤、意向分級 | 分數、理由、摘要 |
| 成交層 | 轉銷售 + 人工確認 | 工單、成交／失單回寫 |

## 主路徑

```text
客戶訊息（個微／企微）
  → 維護訊息（歸檔／標籤／待辦）
  → 識別意向（熱度／意圖／理由）
  → 轉交銷售（工單 + 摘要）
  → 人工確認 → 成交
  → 結果回寫客戶池
```

## 商機與邊界

- **商機**：席位費、行業代運營、與 CRM／FDE 專案捆綁。  
- **邊界**：人在環；禁灰產全自動；企微優先於個微全自動。  
- **公開連結**：聽後筆記未附穩定產品官網；以架構複用與合規版交付為主。
