01 · Industry

行業畫像：為什麼適合你

西點不是火鍋天天大桌，卻同時有高頻小點與節慶大單——剛好需要「準帳本 + 準推播」。

業態特徵

  • 高頻零食 + 低頻大單：麵包小點週復購；整模生日/節慶蛋糕高客單
  • 強時令：母親節、情人節、中秋、聖誕、畢業季 → 預購高峰
  • 高度 LINE 化：預購常走官方帳號；集點從紙本→LINE
  • 多店 / 加盟常見：中央廚房 + 多門市取貨，要統一會員
  • 損耗敏感：亂發券傷毛利 → 需要人審 AI，不是狂撒

定位一句話

不是「又一個火鍋 Ocard」，而是西點連鎖的「點算準 + 節慶召回 + 預購/取貨一體」中台。

與 Ocard 有重疊（LINE 會員），但更吃重：預購、禮券、多店對帳、節慶控成本。

04 · Product

產品清單

三層：帳本（SoR）→ LINE 觸達 → AI 決策。AI 不直接改餘額。

帳本（必做）

  • 消費給點（可排除已折價/禮券）
  • 品類倍率（小點 1x / 整模 1.5x）
  • 禮券、提貨券獨立券種
  • 預購付清才入帳、取消沖正
  • 跨店同一 member_id
  • 點/券過期與流水審計

規則模板（P0 預設）

  • 每滿 NT$50 → 1 點（可調）
  • 10 點 → NT$50 或指定小點
  • 綁 LINE → 入會禮 1 點/小可頌券
  • 生日：切片免費 或 整模 9 折（二選一防疊）
  • 銀/金級：近 12 月消費額

OMO 資料流

POS→ 線上商城→ 外送

三通路訂單 → 統一 member + ledger → LINE / 取貨碼 / 核銷。

05 · Channels

POS + 線上商城 + 外送

三條進單通道，一本會員帳。我們不取代 POS / 開店平台 / 外送平台，只做會員帳本與決策中台。

通路 1

POS 門市

  • 結帳掃會員碼 / 手機號 → 給點
  • 現場兌獎、核銷券
  • 預購取貨掃碼確認
  • 多店共用同一 member

對接：POS Webhook / 日結檔 / 中間件

通路 2

線上商城

  • 官網 / App / 91 / SHOPLINE 等
  • 登入即綁會員，下單給點
  • 購物車點數折抵、券折
  • 禮盒預購、鑑賞期後入帳

對接：訂單 Webhook + 會員 SSO/綁定

通路 3

外送

  • 自外送 / 品牌外送：訂單可帶 member，正常給點
  • 第三方（Uber/foodpanda 等）：平台常吃掉會員身份
  • 策略：外送單附「回門市/LINE 補登」或僅品牌自外送給點
  • 外送運費是否給點：規則可關

對接：自有外送 API；三方靠補登/券引導

統一訂單模型（入帳必填）

欄位說明

order_id來源系統唯一單號（冪等鍵）
member_id可空（訪客）；有則給點
amount_eligible可計點金額（已扣券/運費策略後）

statuspaid / cancelled / refunded → 沖正
場景給點？兌獎/用券？注意
POS 門市是（綁會員）是主力；教育掃碼 15 分
線上商城自取付清後 / 鑑賞期後是取貨提醒走 LINE 交易
線上商城宅配出貨或簽收後（可配置）是退貨沖正
品牌自外送是（若已登入）是運費是否計點可關
第三方外送預設否或僅補登難用 LINE 券導回官網/門市

架構原則

  • POS / 商城 / 外送 = 訂單來源
  • 本系統 = 會員帳本 + 券 + LINE + AI
  • 同一人：手機 / LINE user / 商城帳號綁定
  • 失敗可補登，必有操作者審計

與誰共存（不搶位置）

  • POS：原廠或既有收銀
  • 線上：91APP / SHOPLINE / 自建官網
  • 外送：Uber Eats / foodpanda / 自外送
  • 話術：「開店與外送繼續用您的；我們讓三通路點算在同一本帳。」

實施分期（含三通路）

  1. P0 · POS門市給點 + LINE 到帳（必做，價值最快）
  2. P1 · 線上商城訂單 Webhook + 購物車用點/券
  3. P2 · 自外送帶 member 的外送單入帳
  4. P3 · 三方外送補登流程 + AI 導流券（不強求平台原生積分）
報價提示：三通路對接工時不同。POS 基礎含在實施費；第二個商城平台、自外送 API、三方特殊串接可列「通路加購」。Growth 年費含帳本能力，不含無限客製接三家 POS。

07 · AI Decision

AI 決策系統

Growth 加送基礎版：只出草稿與解釋，預設人審。Pro 才開守衛內自動。

決策輸入輸出（預設草稿）

櫃量出清門市 + SKU 殘量（可後接）小範圍 1.5x 點或小券

加送範圍（含）

  • RFM / 沉默分層
  • 活動草稿 + 原因說明
  • 人審後才發送

  • 自動發券 / 排隊推播

  • 外接 CDP
護欄示例：單客月行銷折讓 ≤ 客單 15%；節慶券庫存上限；AI 永不直接改積分餘額。

09 · Sales Script

話術（一頁 Script）

A. 開場 30 秒

西點客人在門市 POS、官網商城、外送都會下單，但會員常斷成三本帳。 多店點數對不齊、節慶全民折砍毛利。 我們做：三通路進同一帳本、LINE 推播、AI 建議該推誰——您點頭才發券。 不換 POS、不換商城、不換外送平台，30 天先跑通門市給點與到帳。 會員核心約 Ocard 七成，Growth 加送 AI 決策基礎。

B. 資格篩選

問理想若否
幾間門市？≥3 或有預購/禮券建議官方集點，留聯絡
有 POS / 能出訂單？有或可 Webhook/日結半自動補登方案
主要用 LINE？是可做，價值打七折
誰拍板？老闆/營運/行銷約決策者同場

C. 方案三刀（各 20 秒）

  • 帳本：所有門市、預購付清、禮券核銷，同一流水。
  • LINE：到帳、取貨提醒走交易；活動走 opt-in。
  • AI：只出「給誰、什麼券」草稿，預設您審核。

Demo 順序：給點 → 到帳推播 → 流水 → AI 母親節名單草稿 → 核准 → 發券 → 核銷報表。

D. 價格話術

  • 開價：「會員核心對齊約 七成，且 AI 基礎直接加送。」
  • 對方拿 Ocard 全包比：「請拆開點餐/商城；比會員+推播+決策。」
  • 再砍：「年費已含 AI；再砍改 Starter 範圍，不砍壞帳本 SLA。」
  • 要自動 AI：「那是 Pro，不是加送範圍。」

E. 逼定下一步

① 技術對接 45 分（POS/IT + 營運）　或　② 下一檔節慶试点（人審發送 + 復盤核銷）

我們不跟您搶開店系統；我們讓點算得準、節慶推得準、帳對得完。

13 · Case

案例（示意 · 可換真客戶）

「北中南 12 店烘焙」· POS + 官網 + 外送

痛點

門市集點、官網另一套、外送沒會員；母親節全民 8 折；跨店爭議多。

做法

P0 POS 統一帳本 → P1 官網 Webhook 給點/用券 → 自外送帶 member；AI 節慶人審名單；三方外送只做 LINE 導流券。

結果（示意）

有 member 訂單占比↑；跨店爭議→0；節慶折讓可控；分通路報表可看。

15 · Objections

異議處理表

異議回應要點
已有 LINE 官方集點單店夠；跨店、商城、外送、節慶分眾才需要統一帳本。
跟 Ocard 一樣？Ocard 擅餐飲；我們強在 POS+商城+外送同一帳 + AI 人審。核心約 7 成+AI 加送。
要換 POS / 商城嗎？不用。原 POS、91/SHOPLINE、外送平台都留；我們只接訂單進帳本。
外送平台不給會員資料三方外送預設不硬給點；用 LINE 導流券回門市/官網；自外送可完整給點。
門市太忙POS 自動給點；最多掃碼一步；補登有審計。
AI 亂送會虧預設草稿+人審；折讓 cap；不直接改餘額。
已有 91APP賣場繼續用 91；我們當積分與決策中台，可共存。
不要裝 AppLINE 綁定即可，不需新 App。
三通路一次接不完P0 POS → P1 商城 → P2 外送；先讓門市帳準就有感。
很貴年費含帳本能力；第二套商城/自外送可列通路加購；可比亂折成本。
資料外流租戶隔離、opt-in、所有權歸品牌、可匯出。
